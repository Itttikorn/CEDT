HW 7: Generative AI

Large language models (LLMs), like ChatGPT and Gemini, have emerged as transformative tools in today's world. These sophisticated models are usually trained on diverse enormous datasets and tasks, which make them achieve the human-level capability on the trained task. Beyond the in-domain performances, LLMs demonstrate exceptional results in unseen data and tasks using only prompt engineering techniques. Prompt engineering involves designing the prompts that instruct the model to perform a specific task or influence a beneficial behavior without retraining the models. The low complexity of this approach makes it a preferable choice for everyday users or even researchers. Nevertheless, without fine-tuning the model, the prompt engineering performance is usually bounded. To grapple with this challenge, the other branches of approach introduce effective ways to fine-tune these colossal models. In this homework, you will learn how to implement these methods. We separate the homework into two sections: 1. Prompt Engineering and 2. Low-Rank Adapation (LoRA).

In the first section, we introduce prompt engineering methods, which do not involve optimizing the model but rather the prompts. It is the easiest method and something that everyone is doing now. In prompt engineering, the prompts are specifically designed to instruct the model to perform a specific task, influencing their behavior.

The second section revolves around methods that optimize the models or parameters. Specifically, you will learn what is Low-rank Adaptation (LoRA).

# Catch transient Gemini errors.
def is_retryable(e) -> bool:
    if retry.if_transient_error(e):
        # Good practice, but probably won't fire with the google-genai SDK
        return True
    elif (isinstance(e, genai.errors.ClientError) and e.code == 429):
        # Catch 429 quota exceeded errors
        return True
    elif (isinstance(e, genai.errors.ServerError) and e.code == 503):
        # Catch 503 model overloaded errors
        return True
    else:
        return False

@retry.Retry(predicate=is_retryable)

Part 2: Low-rank Adapation (LoRA)

With the discovery of scaling properties in deep learning models, several researchers tend to increase model size to achieve emergent properties, especially in the natural language processing (NLP) field. For example, GPT-3 contains 175 billion parameters, making it nearly impossible to fine-tune on limited resources. This trend prevents students like us from adapting these enormous foundation models on a single GPU (or with small resources).

To alleviate this problem, researchers have developed new fine-tuning methods, known as parameter-efficient transfer learning, which allow us to train large models with limited resources. The benefits of these methods extend not only to the training process but also to deployment. After fine-tuning, we only need to save a small number of parameters (the LoRA weights), enabling us to deploy the foundation model to various downstream tasks using minimal storage. One of the prevailing methods is Low Rank Adaptation (LoRA).

Another popular option is prompt tuning, where we only train special tokens that are prepended to the input. However, this is not the focus of this homework.

In this section, we will introduce traditional fine-tuning and LoRA.

class IMDbPromptTuningDataset(Dataset):
    def __init__(self, dataset, tokenizer, max_length=512):
        self.dataset = dataset
        self.tokenizer = tokenizer
        self.max_length = max_length

    def __len__(self):
        return len(self.dataset)

    def __getitem__(self, idx):
        text = self.dataset[idx]["text"]
        label = self.dataset[idx]["label"]

        encoding = self.tokenizer(
            text,
            truncation=True,
            padding="max_length",
            max_length=self.max_length,
            return_tensors="pt"
        )

        return {
            "input_ids": encoding["input_ids"].squeeze(),
            "attention_mask": encoding["attention_mask"].squeeze(),
            "labels": torch.tensor(label, dtype=torch.long)
        }
